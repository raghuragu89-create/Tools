🔧 Complete Manual Deployment (Step-by-Step)

Follow this if you want to deploy the AI backend yourself without the tool's auto-deploy.

Prerequisites

  • AWS CLI installed and configured
  • Admin credentials for an AWS account
  • Bedrock model access enabled (see Option 1, Step 3 above)

Step 1: Verify AWS Access

# Check you're authenticated
aws sts get-caller-identity --region us-east-1

# Expected output:
# {
#     "UserId": "AROAXXXXXXXXXXXXXXXXX:youralias",
#     "Account": "123456789012",
#     "Arn": "arn:aws:sts::123456789012:assumed-role/Admin/youralias"
# }
If this fails: Run ada credentials update --account=YOUR_ACCOUNT_ID --role=Admin --once

Step 2: Create the Lambda Function

2a. Create IAM Role

# Create trust policy file
cat > trust-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"Service": "lambda.amazonaws.com"},
    "Action": "sts:AssumeRole"
  }]
}
EOF

# Create the role
aws iam create-role \
  --role-name testrail-analyzer-lambda-role \
  --assume-role-policy-document file://trust-policy.json

# Attach basic Lambda execution policy
aws iam attach-role-policy \
  --role-name testrail-analyzer-lambda-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole

# Create Bedrock access policy
cat > bedrock-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["bedrock:InvokeModel", "bedrock:InvokeModelWithResponseStream"],
    "Resource": "*"
  }]
}
EOF

aws iam put-role-policy \
  --role-name testrail-analyzer-lambda-role \
  --policy-name BedrockInvoke \
  --policy-document file://bedrock-policy.json

# Wait for role to propagate
echo "Waiting 10 seconds for IAM role to propagate..."
sleep 10

2b. Create Lambda Function Code

# Create the Lambda handler file
cat > index.py << 'EOF'
import json, boto3, os, re

bedrock = boto3.client("bedrock-runtime", region_name="us-east-1")
MODEL = os.environ.get("MODEL_ID", "anthropic.claude-sonnet-4-20250514-v1:0")

def handler(event, context):
    try:
        body = json.loads(event.get("body", "{}")) if isinstance(event.get("body"), str) else event.get("body", {})

        # Support both formats: direct (system/content) and wrapped (systemPrompt/userPrompt)
        system_prompt = body.get("system") or body.get("systemPrompt", "You are a QA automation expert.")
        user_content = body.get("content") or body.get("userPrompt", "")
        max_tokens = body.get("max_tokens") or body.get("maxTokens", 4096)

        resp = bedrock.invoke_model(
            modelId=MODEL,
            contentType="application/json",
            accept="application/json",
            body=json.dumps({
                "anthropic_version": "bedrock-2023-05-31",
                "max_tokens": int(max_tokens),
                "system": system_prompt,
                "messages": [{"role": "user", "content": user_content}]
            })
        )
        result = json.loads(resp["body"].read())
        text = result.get("content", [{}])[0].get("text", "")

        # Try to parse JSON from response
        try:
            parsed = json.loads(text)
        except Exception:
            m = re.search(r'\[.*\]', text, re.DOTALL)
            if m:
                parsed = json.loads(m.group())
            else:
                parsed = {"raw": text}

        # Wrap in results array if it's a list
        if isinstance(parsed, list):
            response_body = {"results": parsed}
        else:
            response_body = parsed

        return {
            "statusCode": 200,
            "headers": {"Content-Type": "application/json", "Access-Control-Allow-Origin": "*"},
            "body": json.dumps(response_body)
        }
    except Exception as e:
        return {
            "statusCode": 500,
            "headers": {"Content-Type": "application/json"},
            "body": json.dumps({"error": str(e)})
        }
EOF

# Package it
zip function.zip index.py

2c. Deploy Lambda

# Get your account ID
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)

# Create the function
aws lambda create-function \
  --function-name testrail-analyzer \
  --runtime python3.12 \
  --handler index.handler \
  --role arn:aws:iam::${ACCOUNT_ID}:role/testrail-analyzer-lambda-role \
  --zip-file fileb://function.zip \
  --timeout 120 \
  --memory-size 256 \
  --environment "Variables={MODEL_ID=anthropic.claude-sonnet-4-20250514-v1:0}" \
  --region us-east-1

# Verify it was created
aws lambda get-function --function-name testrail-analyzer --region us-east-1 --query "Configuration.FunctionArn"
✅ Lambda created! Now we need API Gateway to expose it as an HTTPS endpoint.

Step 3: Create API Gateway

3a. Create REST API

# Create the API
API_ID=$(aws apigateway create-rest-api \
  --name "testrail-analyzer-api" \
  --description "TestRail Analyzer AI endpoint" \
  --region us-east-1 \
  --query "id" --output text)

echo "API ID: $API_ID"

# Get root resource ID
ROOT_ID=$(aws apigateway get-resources \
  --rest-api-id $API_ID \
  --region us-east-1 \
  --query "items[?path=='/'].id" --output text)

echo "Root Resource ID: $ROOT_ID"

3b. Create /analyze Resource + POST Method

# Create /analyze resource
RESOURCE_ID=$(aws apigateway create-resource \
  --rest-api-id $API_ID \
  --parent-id $ROOT_ID \
  --path-part "analyze" \
  --region us-east-1 \
  --query "id" --output text)

echo "Resource ID: $RESOURCE_ID"

# Create POST method with API key required
aws apigateway put-method \
  --rest-api-id $API_ID \
  --resource-id $RESOURCE_ID \
  --http-method POST \
  --authorization-type NONE \
  --api-key-required \
  --region us-east-1

# Get account ID and region for Lambda ARN
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
LAMBDA_ARN="arn:aws:lambda:us-east-1:${ACCOUNT_ID}:function:testrail-analyzer"

# Create Lambda integration
aws apigateway put-integration \
  --rest-api-id $API_ID \
  --resource-id $RESOURCE_ID \
  --http-method POST \
  --type AWS_PROXY \
  --integration-http-method POST \
  --uri "arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/${LAMBDA_ARN}/invocations" \
  --region us-east-1

3c. Grant API Gateway Permission to Invoke Lambda

aws lambda add-permission \
  --function-name testrail-analyzer \
  --statement-id apigateway-invoke \
  --action lambda:InvokeFunction \
  --principal apigateway.amazonaws.com \
  --source-arn "arn:aws:execute-api:us-east-1:${ACCOUNT_ID}:${API_ID}/*" \
  --region us-east-1

3d. Deploy API to "prod" Stage

# Create deployment
aws apigateway create-deployment \
  --rest-api-id $API_ID \
  --stage-name prod \
  --region us-east-1

echo ""
echo "========================================="
echo "YOUR ENDPOINT URL:"
echo "https://${API_ID}.execute-api.us-east-1.amazonaws.com/prod/analyze"
echo "========================================="
✅ API Gateway deployed! Your endpoint URL is ready. Now create an API key.

Step 4: Create API Key + Usage Plan

4a. Create Usage Plan (rate limiting)

# Create usage plan with 10,000 requests/day
PLAN_ID=$(aws apigateway create-usage-plan \
  --name "testrail-analyzer-plan" \
  --throttle "rateLimit=20,burstLimit=30" \
  --quota "limit=10000,period=DAY" \
  --api-stages "apiId=${API_ID},stage=prod" \
  --region us-east-1 \
  --query "id" --output text)

echo "Usage Plan ID: $PLAN_ID"

4b. Create API Key

# Create the key
KEY_ID=$(aws apigateway create-api-key \
  --name "testrail-analyzer-key" \
  --enabled \
  --region us-east-1 \
  --query "id" --output text)

echo "API Key ID: $KEY_ID"

# Associate key with usage plan
aws apigateway create-usage-plan-key \
  --usage-plan-id $PLAN_ID \
  --key-id $KEY_ID \
  --key-type API_KEY \
  --region us-east-1

# Get the actual API key value
API_KEY=$(aws apigateway get-api-key \
  --api-key $KEY_ID \
  --include-value \
  --region us-east-1 \
  --query "value" --output text)

echo ""
echo "========================================="
echo "YOUR API KEY:"
echo "$API_KEY"
echo "========================================="

🔄 Option 3: Shared Backup Endpoint

This requires zero setup. Click "🔄 Use Shared" in the Setup popup.

How it works:
  • Connects to a team-managed AWS account
  • Shared across all users of the tool
  • Daily quota: 5,000 test cases
  • Quota resets at 12:00 AM IST (6:30 PM UTC)
⚠️ When the limit is reached:
The tool will show: "DAILY LIMIT REACHED (5,000 TC/day). Please try again tomorrow after 12:00 AM IST."
It automatically switches to heuristic-only mode — your analysis still completes, just without AI deep analysis for remaining TCs.

Upgrading from Shared to Personal

  1. Click ❌ Clear in the AI ENGINE section
  2. Click ⚡ Setup My AI
  3. Follow Option 1 or 2 to connect your personal endpoint

💬 FAQ

Q: Is the shared endpoint secure?

Yes. It uses API key authentication. The key is stored in your Windows Credential Manager (via keyring), not in plain text files.

Q: Can I share my personal endpoint with teammates?

Yes — give them your Endpoint URL + API Key. They can paste it in Option 2 (Manual Entry). But they'll share your 10,000/day quota.

Q: What happens if AI is unavailable?

The tool automatically falls back to heuristic-only mode. It still produces results using keyword matching, step analysis, and pattern detection — just without the AI deep reasoning layer.

Q: How do I delete my personal stack?

aws cloudformation delete-stack \
  --stack-name testrail-analyzer-youralias \
  --region us-east-1

This removes Lambda, API Gateway, and all associated resources. Zero residual cost.

Q: Can I use a different AI model?

Yes. Change the MODEL_ID environment variable on your Lambda function:

aws lambda update-function-configuration \
  --function-name testrail-analyzer \
  --environment "Variables={MODEL_ID=anthropic.claude-3-haiku-20240307-v1:0}" \
  --region us-east-1

Supported: claude-sonnet-4-20250514-v1:0 (best), claude-3-haiku (faster/cheaper)
